# Surface docs-site Console Kit adoption handoff

## Approach

Surface uses the package name `@kontourai/console-kit` but does not add it to `package.json`. This avoids making standalone Surface installs depend on a sibling checkout outside the Surface repository.

The docs build runs `scripts/sync-console-kit-assets.mjs` before generating pages. The script resolves token assets in this order:

1. installed `node_modules/@kontourai/console-kit`
2. sibling workspace checkout `../console-ui`

Both source paths validate that `package.json` has `name: "@kontourai/console-kit"` before copying. The script copies the resolved package token assets into the generated docs site at:

```text
docs-site/vendor/console-kit/tokens/
```

Generated pages link the shared token stylesheet before the Surface stylesheet:

```html
<link rel="stylesheet" href="vendor/console-kit/tokens/index.css">
<link rel="stylesheet" href="styles.css">
```

This keeps the static docs-site deployable without symlinks or a bundler while preserving a package-name-validated token contract.

## Theme decision

Generated pages now use:

```html
<html lang="en" class="theme-surface">
```

Surface keeps automatic light/dark via `prefers-color-scheme`; it does not force `data-theme`. The docs site preserves the existing editorial Surface identity, including the serif typography, warm light skin, dark green skin, terrain texture, and large hero treatment.

Surface-specific identity colors are authored as `--k-*` overrides in `docs-site/styles.css` generated by `scripts/build-pages-site.mjs`. Local presentation aliases such as `--surface-panel`, `--surface-brand-glow`, and `--surface-accent-secondary` are derived from those `--k-*` values.

## Files changed

- `surface/package.json`
  - Adds `sync:console-kit` and `check:console-kit-assets`.
  - Updates `docs:build` and `verify` to sync and validate vendored token assets.
- `surface/scripts/sync-console-kit-assets.mjs`
  - Copies token assets into `docs-site/vendor/console-kit/tokens/`.
  - Resolves an installed `@kontourai/console-kit` first, then the sibling `../console-ui` package.
  - Validates the package name before copying.
  - Supports `--check` drift detection.
  - Rejects symlinked vendored assets.
- `surface/scripts/build-pages-site.mjs`
  - Cleans stale ignored `docs-site/*.html` and `docs-site/styles.css` before regenerating.
  - Adds `theme-surface` to generated pages.
  - Links Console Kit tokens before `styles.css`.
  - Replaces legacy local `--bg`, `--panel`, `--accent`, etc. with `--k-*` token usage and Surface-local derived aliases.

## Parity gaps / assumptions

- Surface docs intentionally do not adopt React or custom-element primitives in this pass; the scope is token/theme adoption for the generated static docs site.
- `docs-site/` is ignored/generated in Surface, so the committed source of truth is the generator plus the sync script.
- Surface keeps its own visual identity rather than adopting `theme-console`, `theme-flow`, or survey styling.
- Surface standalone installs do not require the sibling Console Kit checkout; docs token sync requires either an installed package or `../console-ui`.
